# Ralli 랜딩 페이지 SEO 보강 작업 로그

> 작업일: 2026-07-24
> 브랜치: `feature/ralli-seo` → `develop` ([PR #61](https://github.com/qlrogo91lp/yj-blog/pull/61))

## 1. 문제 제기

`/apps/ralli` 랜딩 페이지의 SEO 상태를 점검해달라는 요청에서 시작했다. 페이지 전용 `title`/`description`은 있었지만, 그 외 항목을 확인한 결과 세 가지가 비어 있었다.

- **OG/Twitter 이미지** — 페이지 전용 지정이 없어 루트 레이아웃의 범용 `/og-default.png`로 폴백되고 있었다
- **canonical URL** — 미지정
- **구조화 데이터(JSON-LD)** — 없음 → 앱스토어 링크 등을 검색 결과 리치 스니펫으로 노출할 수 없는 상태

나머지(sitemap 포함, robots 허용, `h1`, 이미지 alt, `metadataBase`)는 이미 정상이었다.

## 2. 적용한 변경

- [src/app/(main)/apps/ralli/page.tsx](../../src/app/(main)/apps/ralli/page.tsx)
  - `alternates.canonical: '/apps/ralli'` 추가
  - `openGraph`/`twitter`에 Ralli 앱 아이콘(`/ralli/icon1.png`, 1024×1024) 기반 전용 이미지·제목·설명 지정
- [src/app/(main)/apps/ralli/_components/ralli-json-ld.tsx](../../src/app/(main)/apps/ralli/_components/ralli-json-ld.tsx) (신규)
  - `SoftwareApplication` JSON-LD 스키마 — 이름, 카테고리(`SportsApplication`), OS(`iOS, watchOS`), 앱스토어 다운로드 링크, 무료(`offers`) 정보 포함

## 3. 검증

- `npm run lint` — 에러 0 (기존 경고만, 회귀 없음)
- 포트 3000이 다른 프로젝트(학온)에 점유되어 있어 3210 포트로 dev 서버를 임시 기동, 브라우저에서 렌더된 `<head>` 직접 확인
  - `canonical` → `https://yjlogs.com/apps/ralli`
  - `og:image`/`twitter:image` → `https://yjlogs.com/ralli/icon1.png`
  - `application/ld+json` 파싱 결과 스키마 필드 전부 정상 출력
  - `h1`("Ralli") 등 기존 렌더링 회귀 없음

## 4. 남은 과제

- **OG 이미지가 정사각형** — 지금은 앱 아이콘(1024×1024)을 그대로 써서 카카오톡/트위터의 1200×630 landscape 카드에서 레터박싱될 수 있다. 스크린샷을 활용한 전용 가로형 OG 이미지 제작을 고려할 수 있다.
- **`<html lang="ko">`와 영문 콘텐츠 불일치** — Ralli 페이지 본문(태그라인·설명)이 전부 영어인데 사이트 전역 `lang`은 `ko`로 고정되어 있다. 검색엔진 언어 신호 혼동 가능성이 있어 필요 시 해당 섹션에 `lang="en"` 명시를 검토한다.
