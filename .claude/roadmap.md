## 홈 페이지 (`/`) 개편

### 현재 문제

현재 `/`와 `/posts`가 거의 동일하다 (글 목록 + 뷰 토글). 차이는 `/posts`에 카테고리 필터·페이지네이션이 더 있다는 것뿐. 사실상 홈이 `/posts`의 축약판이라 **역할이 중복**된다.

### 역할 재정의

| 페이지 | 역할 |
|--------|------|
| `/` (홈) | **랜딩 페이지**. 블로그를 처음 방문한 사람에게 *누가, 무엇을, 어떻게* 쓰는지 보여주는 곳 |
| `/posts` | **아카이브**. 모든 글을 필터·페이지네이션으로 탐색하는 곳 |

### 구성 섹션 후보

| 섹션 | 우선순위 | 설명 |
|------|---------|------|
| Hero / 소개 | ⭐⭐⭐ | 이름, 한 줄 자기소개, 블로그 주제. 사진/아바타 또는 일러스트. CTA 버튼(`글 보러가기` → `/posts`) |
| 최근 글 (Featured) | ⭐⭐⭐ | 최신 글 3~5개를 카드형으로. 전체 목록 X. 하단에 `모든 글 보기` 링크 |
| 카테고리 바로가기 | ⭐⭐ | 주요 카테고리를 칩/태그 형태로. 클릭 시 `/categories/[slug]` 이동 |
| 인기 글 | ⭐⭐ | 조회수 기반 Top 3~5. (통계 시스템이 이미 있으므로 활용 가능) |
| About 섹션 | ⭐ | 사용 기술 스택, 관심사, 소셜 링크(GitHub, Email 등). 길어지면 별도 `/about` 페이지로 분리 |
| Apps / Playground 프로모션 | ⭐ | 글 외에 만든 콘텐츠가 있다는 걸 보여주는 짧은 카드 영역 |

### 단계별 구현 제안

1. ~~**Phase 1 — 최소 랜딩**: Hero + 최근 글(5개) + 모든 글 보기 링크. 가장 먼저 `/posts`와 차별화.~~ ✅
2. **Phase 2 — 탐색 보조**: 카테고리 바로가기 + 인기 글 섹션. 통계 쿼리 재사용.
3. **Phase 3 — 정체성 강화**: About 섹션 / `/about` 페이지, Apps·Playground 프로모션 카드.

### 기술 메모

- Hero·About 등 정적 텍스트는 콘텐츠 변경 빈도가 낮으므로 상수 파일(`src/app/(main)/_constants/profile.ts` 등)에 두는 것을 우선 검토. 추후 `/admin/settings`에서 편집 가능하게 한다면 DB로 이전.
- 인기 글은 기존 통계 쿼리가 있다면 그대로 활용. 없으면 별도 캐시 태그(`CACHE_TAGS.popularPosts` 등)로 `unstable_cache` 처리.
- `/`는 Server Component 유지. 인터랙션이 필요한 부분만 `_actions`로 분리.
- `_actions/view-toggle-action.tsx`, `_handlers/post-list-view-handler.tsx`는 더 이상 홈에서 필요 없으므로 `/posts` 전용으로 이동 검토.
