사용자가 언급한 폴더나 파일을 **세세하게 분석**한 뒤, 수정 계획을 `.claude/plan.md`에 작성한다.

## 분석 절차

1. **폴더/파일 탐색**: 언급된 경로의 모든 파일을 읽고 구조를 파악한다
2. **현황 파악**: 각 파일의 역할, 의존성, 문제점을 분석한다
3. **규칙 대조**: `.claude/rules/` 의 컨벤션과 비교해 위반 사항을 식별한다
4. **계획 수립**: 구체적인 수정 방향, 우선순위, 코드 스니펫을 작성한다

## plan.md 작성 형식

`.claude/plan.md` 파일에 아래 구조로 작성한다:

```markdown
# 수정 계획: [대상 폴더/파일]

> 작성일: [오늘 날짜]

## 현재 구조

[현재 폴더/파일 트리]

## 분석 요약

[주요 문제점 또는 개선 포인트 요약]

## 수정 계획

### 1. [항목명]

**현재 코드**
\`\`\`tsx
// 현재 코드 스니펫
\`\`\`

**수정 후**
\`\`\`tsx
// 수정된 코드 스니펫
\`\`\`

**이유**: [수정 이유 설명]

---

### 2. [항목명]

...

## 변경 후 구조

[수정 후 예상 폴더/파일 트리]

## 체크리스트

- [ ] [항목 1]
- [ ] [항목 2]
```

## 주의사항

- plan.md 작성만 한다. 실제 코드 수정은 하지 않는다
- 코드 스니펫은 실제 파일 내용 기반으로 작성한다 (추측 금지)
- 컨벤션 위반, 리팩토링 포인트, 구조 개선 모두 포함한다
- 파일이 많으면 우선순위(High / Medium / Low)를 붙인다
